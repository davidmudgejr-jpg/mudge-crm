# AI System Enhancements — Cost Tracking, Council Briefing, Smart Filtering, Audit Logging

**Date:** 2026-03-13
**Status:** Draft
**Source:** Patterns adapted from Matthew Berman's OpenClaw use cases, tailored for CRE AI system

## Overview

Four enhancements to the existing AI system architecture, implemented as **layered upgrades to existing components** — no new agents, no new infrastructure.

| Enhancement | Component Modified | Purpose |
|---|---|---|
| Per-Call Cost Tracker | Shared utility + supervisor config | Token-level cost visibility across all models |
| Council Briefing | Houston's 6 AM review | Multi-perspective adversarial review of daily briefing |
| Two-Stage Enrichment Filtering | Enricher agent workflow | Eliminate junk before paid API calls |
| JSONL Structured Audit Log | Shared utility + all agents | Machine-readable log of every agent decision |

## 1. Per-Call Cost Tracker

### Purpose

Track every LLM/API call at the individual request level — model, token counts, task type, cost estimate, and duration. Feeds into the Logger agent's reports and enables routing optimization.

### Location

- **Utility:** `/AI-Agents/shared/cost-tracker.py`
- **Config:** Pricing table in `supervisor-config.json`
- **Output:** JSONL entries in the audit log (action type: `llm_call`)

### Log Entry Format

```jsonl
{
  "timestamp": "2026-03-25T06:01:23Z",
  "agent": "enricher",
  "model": "qwen-3.5-20b",
  "provider": "ollama_local",
  "tokens_in": 1240,
  "tokens_out": 380,
  "task_type": "contact_classification",
  "description": "Classify LLC registered agent as real person vs role",
  "cost_estimate": 0.00,
  "duration_ms": 1850
}
```

### Pricing Lookup Table

Maintained in `supervisor-config.json` under `"pricing"` key:

```json
{
  "pricing": {
    "anthropic/opus-4.6":    { "input_per_1m": 15.00, "output_per_1m": 75.00 },
    "anthropic/sonnet-4.6":  { "input_per_1m": 3.00,  "output_per_1m": 15.00 },
    "anthropic/haiku-4.5":   { "input_per_1m": 0.80,  "output_per_1m": 4.00 },
    "openai/gpt-4o":         { "input_per_1m": 2.50,  "output_per_1m": 10.00 },
    "openai/gpt-4o-mini":    { "input_per_1m": 0.15,  "output_per_1m": 0.60 },
    "google/gemini-flash":   { "input_per_1m": 0.30,  "output_per_1m": 1.20 },
    "google/gemini-pro":     { "input_per_1m": 1.25,  "output_per_1m": 5.00 },
    "ollama/*":              { "input_per_1m": 0.00,  "output_per_1m": 0.00 },
    "_default":              { "input_per_1m": 1.00,  "output_per_1m": 3.00 }
  }
}
```

### Cost Formula

```
cost = (tokens_in / 1,000,000) × input_price + (tokens_out / 1,000,000) × output_price
```

### Reports (Generated by Logger Agent)

Reports can be triggered on-demand or run weekly. Support filters: `--days N`, `--model X`, `--task-type Y`, `--agent Z`.

**Report types:**
- **Overall summary:** Total spend, total calls, average cost per call
- **By model:** Which models are consuming the budget
- **By task type:** Which operations cost the most
- **By agent:** Per-agent spend breakdown
- **Daily trend:** Last 10 days with 7-day moving average
- **Routing suggestions:** Flag expensive models on simple tasks (e.g., Opus on yes/no classification → recommend Haiku or local Qwen)

### Key Rules

- Local Ollama calls are logged with `cost_estimate: 0.00` — complete visibility into system activity, not just spend
- Pricing table is updated by the Scout agent when new model pricing is announced
- Cost tracker function is synchronous — must not slow down agent operations. Writes are append-only to JSONL (fast I/O)

## 2. Council Briefing — Multi-Perspective Morning Review

### Purpose

Upgrade Houston's daily 6 AM review from a single-pass summary to a **3-phase adversarial council** that catches blind spots, challenges assumptions, and surfaces opportunities a single perspective would miss.

### Current Flow (Being Replaced)

```
Agent logs → Claude reads → Single briefing → Houston + Telegram
```

### New Flow

```
Agent logs + CRM data + market signals
    ↓
Phase 1: Lead Analyst (Claude Opus — Houston)
    ↓ draft briefing with scored recommendations
Phase 2: Council Review (3× Claude Sonnet — parallel)
    ├── DealHunter
    ├── RevenueGuardian
    └── MarketSkeptic
    ↓ each reviews + critiques + proposes additions
Phase 3: Houston Reconciles
    ↓ final ranked briefing with noted disagreements
Output → Houston (team) + Telegram (David, full ops version)
```

### Phase 1: Lead Analyst (Houston — Claude Opus)

Same role Houston already plays. Reads all overnight data:
- Agent logs (enrichment counts, signals found, outreach queued, errors)
- Sandbox activity (pending approvals, rejections, promotions)
- CRM changes (new contacts, updated deals, stale records)
- Market signals from Researcher
- Escalations from Tier 2

**Output format — structured JSON:**

```json
{
  "outlook_score": 72,
  "recommendations": [
    {
      "id": 1,
      "title": "Follow up with Inland Logistics Group — expansion signal",
      "description": "Researcher found hiring posts + new warehouse lease filing in Fontana. Contact enriched with 85 confidence. This matches David's target profile.",
      "evidence": ["researcher_signal_2026-03-24_017", "enricher_contact_489"],
      "impact": 80,
      "effort": 20,
      "confidence": 75,
      "category": "opportunity"
    }
  ]
}
```

Categories: `opportunity`, `risk`, `pipeline_health`, `system_improvement`, `action_required`

### Phase 2: Council Reviewers (3× Claude Sonnet, Parallel)

Each reviewer receives: Houston's draft JSON + raw overnight log data.

#### DealHunter
**Lens:** "What opportunities are we missing?"
**Prompt focus:**
- Look for signals Houston underweighted or buried in the log noise
- Flag contacts/properties that should be higher priority based on deal potential
- Ask "why aren't we chasing X?" when data suggests opportunity
- Propose up to 2 new recommendations Houston missed
- Bias toward action — surface things worth pursuing even at lower confidence

#### RevenueGuardian
**Lens:** "Show me the money — what actually closes?"
**Prompt focus:**
- Challenge timeline assumptions on deals — is the close date realistic?
- Flag pipeline items with stale activity (no touch in 14+ days)
- Ask "what's the cost of inaction?" on high-value items
- Verify that effort scores reflect real work required, not wishful thinking
- Focus on near-term revenue impact (30-60 day horizon)

#### MarketSkeptic
**Lens:** "What's wrong with this data?"
**Prompt focus:**
- Question confidence scores — is the enrichment data actually reliable?
- Flag signals that look too clean or too convenient
- Challenge market assumptions ("Is Fontana warehouse demand really growing, or is this one data point?")
- Ask "are we pattern-matching noise?" when correlations are thin
- Check for recency bias — are we overweighting yesterday's signal vs. the trend?

**Each reviewer outputs:**

```json
{
  "reviewer": "DealHunter",
  "recommendation_reviews": [
    {
      "id": 1,
      "verdict": "support",
      "score_adjustment": { "impact": +5, "confidence": -10 },
      "comment": "Strong signal but confidence should be lower — only one source confirms expansion."
    }
  ],
  "new_recommendations": [
    {
      "title": "Revisit dormant contact: Pacific Industrial Partners",
      "description": "Last touch was 45 days ago but they had 3 properties in pipeline. Worth a check-in.",
      "impact": 60,
      "effort": 10,
      "confidence": 65
    }
  ]
}
```

Verdicts: `support` (agree), `revise` (agree with changes), `reject` (remove from briefing)

### Phase 3: Houston Reconciles (Claude Opus)

Houston receives all three reviews and produces the final briefing.

**Ranking formula:**
```
final_impact = avg(houston_impact, reviewer_adjusted_impacts)
final_confidence = avg(houston_confidence, reviewer_adjusted_confidences)
final_effort = avg(houston_effort, reviewer_adjusted_efforts)

priority = (final_impact × 0.4) + (final_confidence × 0.35) + ((100 - final_effort) × 0.25)
```

**Disagreement handling:**
- If all 3 reviewers `support` → high conviction, note consensus
- If 2 support, 1 rejects → include with caveat noting the dissent
- If 2+ reject → drop from briefing, log reasoning
- New recommendations from reviewers are scored and ranked alongside originals

**Hard constraints:**
- No recommendation can say "send outreach now" or trigger any external action
- All actions require David's explicit approval
- "Publish", "post", "email", "send" verbs are stripped from recommendations

**Output:** Same dual-channel format as today:
- **Houston (team):** Top 3-5 actionable items, market intel, quick stats
- **Telegram (David):** Full ranked recommendations with council notes, disagreements, system health, reverse prompts

### Cost Impact

- Phase 1: Already happening (no additional cost)
- Phase 2: 3× Sonnet calls ≈ ~$0.05-0.10 each = $0.15-0.30/day
- Phase 3: 1× Opus call (slightly longer prompt) ≈ ~$0.05-0.10 additional
- **Total: ~$0.20-0.40/day ≈ $6-12/month**

### Council Audit Trail

Full council trace stored in `/AI-Agents/logs/council/YYYY-MM-DD.json`:
```json
{
  "date": "2026-03-25",
  "phase1_draft": { ... },
  "phase2_reviews": {
    "DealHunter": { ... },
    "RevenueGuardian": { ... },
    "MarketSkeptic": { ... }
  },
  "phase3_final": { ... },
  "models_used": {
    "phase1": "claude-opus-4-6",
    "phase2": "claude-sonnet-4-6",
    "phase3": "claude-opus-4-6"
  },
  "total_cost": 0.37
}
```

This lets Houston review past council sessions during self-improvement — "Did the MarketSkeptic's warning about that Fontana signal turn out to be right? Adjust weighting accordingly."

## 3. Two-Stage Enrichment Filtering

### Purpose

Add a **Stage 0 pre-filter** to the Enricher agent that uses Qwen locally (free) to eliminate records that shouldn't be enriched before any paid API calls are made.

### Current Enricher Flow

```
LLC record → Open Corporates → White Pages → BeenVerified → Confidence Score → NeverBounce → Sandbox
```

Every record hits paid APIs regardless of quality.

### New Enricher Flow

```
LLC record
    ↓
Stage 0: Local Pre-Filter (Qwen — FREE)
    ├── PASS → Continue to enrichment pipeline
    └── SKIP → Log reason, move to next record
    ↓
Stage 1: External Enrichment (existing pipeline, unchanged)
    Open Corporates → White Pages → BeenVerified → Confidence Score → NeverBounce → Sandbox
```

### Pre-Filter Rules

Maintained in `/AI-Agents/enricher/pre-filter-rules.json` so Claude can tune them during self-improvement without rewriting agent code:

```json
{
  "version": "1.0",
  "updated_by": "chief_of_staff",
  "updated_at": "2026-03-25T06:00:00Z",
  "rules": {
    "required_fields": {
      "enabled": true,
      "description": "Must have entity name + at least one of: address, state, entity number",
      "minimum_fields": ["entity_name"],
      "plus_one_of": ["address", "state", "entity_number"]
    },
    "property_type_filter": {
      "enabled": true,
      "include": ["industrial", "office", "retail", "multifamily", "mixed_use", "land_commercial"],
      "exclude": ["single_family_residential", "vacant_rural", "religious", "government", "agricultural"]
    },
    "geography_filter": {
      "enabled": true,
      "target_markets": ["Inland Empire", "San Bernardino County", "Riverside County"],
      "adjacent_markets": ["Los Angeles County", "Orange County"],
      "exclude_states": []
    },
    "duplicate_detection": {
      "enabled": true,
      "match_fields": ["entity_name", "registered_agent_name"],
      "stale_threshold_days": 90,
      "description": "Skip if high-confidence match exists in IE CRM and data is <90 days old"
    },
    "junk_entity_filter": {
      "enabled": true,
      "entity_name_contains": ["DISSOLVED", "CANCELLED", "INACTIVE", "REVOKED", "SUSPENDED"],
      "registered_agent_is_service": ["CT CORPORATION", "CSC", "CORPORATION SERVICE", "NATIONAL REGISTERED AGENTS", "LEGALINC", "COGENCY GLOBAL", "UNITED AGENT GROUP"]
    }
  }
}
```

### Filter Logic (Evaluated in Order)

1. **Required fields** → Missing entity name? → SKIP (`reason: missing_required_fields`)
2. **Junk entity filter** → Name contains DISSOLVED/CANCELLED? Registered agent is CT Corp? → SKIP (`reason: junk_entity`)
3. **Property type filter** → Residential/government/religious? → SKIP (`reason: property_type_excluded`)
4. **Geography filter** → Outside target/adjacent markets? → SKIP (`reason: geography_outside_target`)
5. **Duplicate detection** → Fresh high-confidence match exists? → SKIP (`reason: duplicate_fresh`)
6. All filters pass → **ENRICH**

### Audit Integration

Every skip is logged to the JSONL audit log:
```jsonl
{
  "ts": "2026-03-25T14:23:01Z",
  "agent": "enricher",
  "action": "pre_filter_skip",
  "entity": "ACME RESIDENTIAL LLC",
  "reason": "property_type_excluded",
  "details": { "detected_type": "single_family_residential", "rule": "property_type_filter" }
}
```

### Self-Improvement Hook

Houston reviews pre-filter stats weekly:
- If skip rate is >90% → filters may be too aggressive, review rules
- If skip rate is <40% → filters may be too loose, tighten geography or property type
- If a skipped entity later appears as a high-value opportunity → adjust the rule that caught it

Filter rule changes follow the same versioning protocol as agent instruction files — previous versions saved, rollback available.

### Expected Impact

At scale (processing hundreds of LLCs per week):
- **60-80% of records filtered before any paid API call**
- NeverBounce savings: ~$0.008 × filtered_count per batch
- White Pages/BeenVerified savings: ~$0.02-0.05 × filtered_count per batch
- **Net: Enrichment pipeline runs faster and cheaper on higher-quality candidates**

## 4. JSONL Structured Audit Log

### Purpose

Machine-readable, append-only log of every agent decision. Separate from the Logger's human-readable `.md` reports. Feeds the cost tracker, enables Houston's pattern analysis, and provides a grep-friendly audit trail.

### Location

```
/AI-Agents/logs/audit/
├── 2026-03-25.jsonl
├── 2026-03-24.jsonl
├── 2026-03-23.jsonl
└── ...
```

One file per day, rotated at midnight local time.

### Shared Utility

**Location:** `/AI-Agents/shared/audit-log.py`

**Interface:**
```python
def audit(agent: str, action: str, entity: str = None, **details):
    """
    Append a structured entry to today's audit log.

    Args:
        agent: Agent name (enricher, researcher, matcher, scout, logger)
        action: Action type (see standard actions below)
        entity: Entity being acted on (LLC name, contact, property, etc.)
        **details: Additional context (varies by action type)
    """
```

### Standard Action Types

| Action | Used By | Description |
|--------|---------|-------------|
| `llm_call` | All agents | LLM API call (feeds cost tracker) |
| `api_call` | Enricher, Researcher | External API call (Open Corporates, White Pages, etc.) |
| `pre_filter_skip` | Enricher | Record skipped by Stage 0 filter |
| `pre_filter_pass` | Enricher | Record passed Stage 0 filter |
| `sandbox_write` | Enricher, Researcher, Matcher | Data written to sandbox DB |
| `sandbox_promote` | Tier 2 | Sandbox data approved to production |
| `sandbox_reject` | Tier 2 | Sandbox data rejected |
| `outreach_draft` | Matcher | Outreach email drafted |
| `outreach_queued` | Matcher | Outreach queued for review |
| `signal_found` | Researcher | Market signal identified |
| `escalation` | Any agent | Issue escalated to higher tier |
| `heartbeat` | All agents | Agent alive signal |
| `error` | All agents | Error occurred |
| `council_phase` | Houston | Council briefing phase completed |
| `instruction_update` | Houston | Agent instruction file rewritten |

### Entry Examples

```jsonl
{"ts":"2026-03-25T14:23:01Z","agent":"enricher","action":"pre_filter_skip","entity":"ACME RESIDENTIAL LLC","reason":"property_type_excluded","details":{"detected_type":"SFR","rule":"property_type_filter"}}
{"ts":"2026-03-25T14:23:45Z","agent":"enricher","action":"api_call","service":"open_corporates","entity":"INLAND EMPIRE INDUSTRIAL LLC","result":"found","duration_ms":1230,"details":{"registered_agent":"John Smith","state":"CA"}}
{"ts":"2026-03-25T14:24:12Z","agent":"enricher","action":"llm_call","model":"qwen-3.5-20b","provider":"ollama_local","tokens_in":890,"tokens_out":45,"task_type":"contact_classification","cost_estimate":0.00,"duration_ms":920}
{"ts":"2026-03-25T14:24:30Z","agent":"enricher","action":"sandbox_write","entity":"INLAND EMPIRE INDUSTRIAL LLC","confidence":78,"details":{"contact_name":"John Smith","sources":["open_corporates","whitepages"]}}
{"ts":"2026-03-25T06:02:15Z","agent":"houston","action":"council_phase","phase":2,"reviewer":"MarketSkeptic","details":{"recommendations_reviewed":7,"verdicts":{"support":4,"revise":2,"reject":1},"new_recommendations":1}}
```

### Consumers

| Consumer | What it reads | Purpose |
|----------|--------------|---------|
| Cost Tracker | `action: llm_call` | Compute spend per model/agent/task |
| Logger Agent | All actions | Generate daily `.md` summary reports |
| Houston | All actions | Pattern analysis, self-improvement decisions |
| David (manual) | Any | `grep` when investigating issues |
| Pre-filter tuning | `action: pre_filter_skip` | Measure filter effectiveness |

### Retention Policy

| Entry Type | Local Retention | Archive |
|------------|----------------|---------|
| `llm_call`, `api_call`, `heartbeat` | 90 days | Compressed, 1 year |
| `sandbox_write`, `sandbox_promote`, `sandbox_reject` | 1 year | Permanent |
| `outreach_queued`, `outreach_draft` | 1 year | Compliance requirement |
| `escalation` | 1 year | Decision history |
| `error` | 90 days | Compressed, 1 year |
| `council_phase` | 1 year | Self-improvement fuel |
| `pre_filter_skip`, `pre_filter_pass` | 90 days | Compressed, 6 months |

### Performance Requirements

- Audit function must be **non-blocking** — append to file, no network I/O
- One JSONL line per entry, no pretty-printing (saves disk, fast to parse)
- File handle kept open per agent process, flushed every 10 entries or 5 seconds (whichever first)
- Daily rotation handled by supervisor at midnight

## Integration Points

### How These Four Features Connect

```
Every Agent
    │
    ├── calls audit(action, ...) ──→ JSONL Audit Log (/logs/audit/)
    │                                     │
    │                                     ├── Cost Tracker reads llm_call entries
    │                                     │       └── Generates cost reports
    │                                     │
    │                                     └── Logger reads all entries
    │                                             └── Generates daily .md reports
    │
    ├── Enricher calls pre_filter() ──→ Stage 0 Filter
    │       └── Skips logged to audit       └── Rules in pre-filter-rules.json
    │                                             └── Houston tunes rules
    │
    └── Houston 6 AM review
            └── Council Briefing (3-phase)
                    ├── Phase 1: Lead Analyst (Opus)
                    ├── Phase 2: 3× Reviewers (Sonnet, parallel)
                    └── Phase 3: Reconciliation (Opus)
                            └── Trace logged to /logs/council/
```

### Changes to Existing Architecture Files

These additions require updates to:

1. **ARCHITECTURE.md** — Add Council Briefing section under Houston's responsibilities; add Pre-Filter to Enricher workflow
2. **ORCHESTRATION.md** — Add cost tracker utility and audit log utility to shared infrastructure; update supervisor to initialize audit log rotation
3. **OPERATIONS.md** — Update cost tracking section with per-call granularity; add audit log retention policy; add pre-filter rules to self-improvement scope
4. **Enricher agent template** — Add Stage 0 pre-filter step before external API calls
5. **Logger agent template** — Add cost report generation from audit log data
6. **Chief of Staff template** — Add council briefing protocol; add pre-filter rule tuning to self-improvement responsibilities

---

## Build Sequence

These should be built in this order due to dependencies:

1. **JSONL Audit Log** (foundation — everything else writes to it)
2. **Cost Tracker** (reads from audit log)
3. **Two-Stage Filtering** (writes to audit log, independent of cost tracker)
4. **Council Briefing** (writes to audit log, uses cost tracker to log its own spend)
